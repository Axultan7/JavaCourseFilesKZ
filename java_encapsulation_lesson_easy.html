🎯 Encapsulation дегеніміз не?

Encapsulation (Инкапсуляция) - мәліметтер мен әдістерді бір орынға жинақтап, сыртқы қатынауды шектеу принципі. Бұл OOP-тың негізгі принциптерінің бірі.

🔑 Access Modifiers (Қатынау Модификаторлары)

Java-да төрт түрлі access modifier бар. Олар мүшелерге қатынау деңгейін анықтайды.

🔴 private

• Тек сол класста қолжетімді
• Ең қатаң шектеу
• Encapsulation үшін негізгі
• Сыртқы класстардан жасырын

📦 default (package)

• Сол пакетте қолжетімді
• Modifier көрсетпесе default
• Package-level қорғаныс
• Басқа пакеттерден жасырын

🟡 protected

• Пакет + субкластарда қолжетімді
• Inheritance үшін қолданылады
• Мұрагерлік қорғанысы
• Child класстарға ашық

🟢 public

• Барлық жерден қолжетімді
• Ешқандай шектеу жоқ
• API әдістері үшін
• Кез келген класстан қолданыла алады

🔑 Access Modifiers мысалы
public class Person { // Private - тек осы класста private String socialSecurityNumber; // Жеке құпия private double salary; // Жалақы құпия // Default - пакетте ғана String department; // Бөлім ақпараты int employeeId; // Қызметкер ID // Protected - мұрагерлікте де protected String name; // Аты-жөні protected int age; // Жасы // Public - барлық жерден public String email; // Электрондық пошта public String phoneNumber; // Телефон нөмірі }

🔧 Getter және Setter әдістері

Private айнымалыларға қатынау үшін Getter және Setter әдістері қолданылады. Бұл бақыланатын қатынауды қамтамасыз етеді.

📤 Getter әдістері

// Мәнді алу үшін public String getName() { return this.name; } public int getAge() { return this.age; } // Boolean үшін is- префиксі public boolean isAdult() { return this.age >= 18; }

📥 Setter әдістері

// Мәнді орнату үшін public void setName(String name) { if (name != null && !name.trim().isEmpty()) { this.name = name.trim(); } else { throw new IllegalArgumentException("Ат бос болмауы керек"); } } public void setAge(int age) { if (age >= 0 && age <= 150) { this.age = age; } else { throw new IllegalArgumentException("Жас 0-150 арасында болуы керек"); } }

👨‍🎓 Student класс мысалы

public class Student { private String studentId; private String name; private double gpa; // Конструктор public Student(String studentId, String name) { setStudentId(studentId); setName(name); this.gpa = 0.0; } // Validated Setters public void setStudentId(String studentId) { if (studentId == null || !studentId.matches("ST\\d{6}")) { throw new IllegalArgumentException("ID 'ST' + 6 санмен басталуы керек"); } this.studentId = studentId; } public void setGpa(double gpa) { if (gpa < 0.0 || gpa > 4.0) { throw new IllegalArgumentException("GPA 0.0-4.0 арасында болуы керек"); } this.gpa = gpa; } // Getters public String getStudentId() { return studentId; } public String getName() { return name; } public double getGpa() { return gpa; } }

🏠 House класс мысалы

public class House { private String address; private double price; private int rooms; public House(String address, int rooms) { setAddress(address); setRooms(rooms); } public void setPrice(double price) { if (price <= 0) { throw new IllegalArgumentException("Баға оң болуы керек"); } this.price = price; } public void setRooms(int rooms) { if (rooms < 1 || rooms > 20) { throw new IllegalArgumentException("Бөлме саны 1-20 арасында болуы керек"); } this.rooms = rooms; } // Getters public String getAddress() { return address; } public double getPrice() { return price; } public int getRooms() { return rooms; } }

⚠️ Encapsulation-да жиі кездесетін қателер

• Барлық field-терді public етіп қою
• Getter/Setter жазбау немесе дұрыс жазбау
• Validation қоспау
• Access modifier-лерді дұрыс қолданбау

💡 JavaBeans Convention

Java-да Getter/Setter жазу үшін JavaBeans конвенциясы қолданылады:
• Getter: getFieldName() немесе boolean үшін isFieldName()
• Setter: setFieldName(value)
• Field аты camelCase форматында
• IDE-лер автоматты генерациялай алады

🎯 Encapsulation ең жақсы тәжірибелер

🔒 Fields-ті private ету

• Барлық instance variable-дарды private жасаңыз
• Тікелей қатынауды болдырмаңыз
• Controlled access қамтамасыз етіңіз

✅ Validation қосу

• Setter-лерде input тексеру
• Business rules енгізу
• Exception-дар лақтыру
• Қате хабарламаларын түсінікті жасау

🎯 Minimal interface

• Тек қажетті методтарды public ету
• API-ды қарапайым ұстау
• Implementation details жасыру
• Clean interface дизайны

📝 Documentation

• Public методтарға javadoc жазу
• Validation ережелерін түсіндіру
• Exception-дарды документтеу
• Қолдану мысалдарын көрсету

🎓 Практикалық тапсырма - Оңай деңгей

🚗 Car класын жасау тапсырмасы

Бұл тапсырмада сіз қарапайым Car класын жасап, encapsulation принциптерін үйренесіз.

1Private Fields

• brand (String) - көлік маркасы
• model (String) - модель
• year (int) - шыққан жылы
• price (double) - бағасы
• isRunning (boolean) - іске қосылған ба

2Constructor

• brand, model, year параметрлерімен
• price = 0.0, isRunning = false етіп орнату
• Setter әдістерін шақыру validation үшін

3Validated Setters

• brand: null емес, бос емес
• model: null емес, бос емес
• year: 1900-2025 арасында
• price: 0 немесе оң сан

4Getters

• Барлық private field үшін getter
• JavaBeans конвенциясын сақтау
• boolean үшін is- префиксі

5Business Methods

• startEngine() - көлікті іске қосу
• stopEngine() - көлікті тоқтату
• getCarInfo() - толық ақпарат
• getAge() - көліктің жасын есептеу

6Test класы

• Car объектілерін жасау
• Validation тестілеу
• Методтарды сынау
• Нәтижелерді көрсету

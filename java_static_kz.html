⚡ Static дегеніміз не?

Static кілт сөзі — класс мүшелерін (өрістер мен әдістер) объект құрмай-ақ қолдануға мүмкіндік беретін модификатор. Static мүшелер класстың өзіне тиесілі, объектілерге емес.

🔍 Static элементтердің жадыдағы орналасуы:

• Static мүшелер Method Area-да сақталады
• Бағдарлама басталған кезде бір рет ғана құрылады
• Барлық объектілер бір static өрісті пайдаланады
• Бағдарлама аяқталғанға дейін жадыда қалады

📝 Static Синтаксисі

Static өрістің синтаксисі:

[қол жетімділік] static [final] тип айнымалыАты = мәні;
Static әдістің синтаксисі:

[қол жетімділік] static қайтаруТүрі әдісАты(параметрлер) {
    // Код
}
Static блоктың синтаксисі:

static {
    // Инициализация коды
}

⚡ Static элементтерді шақыру:

• КлассАты.staticӨрісі

• new операторы қажет емес

💡 Static Variables-тің ерекшеліктері:

• Барлық объектілер үшін ортақ болып табылады

⚙️ Static Methods (Класстық Әдістер)

⚙️ Static әдістердің мысалдары
// 1. Математикалық операциялар класы class MathUtils { // Static тұрақтылар public static final double PI = 3.14159; public static final double E = 2.71828; // Static әдістер - объект құру қажет емес public static int add(int a, int b) { return a + b; } public static int multiply(int a, int b) { return a * b; } public static double circleArea(double radius) { return PI * radius * radius; } // Өлшемді айналдыру public static double celsiusToFahrenheit(double celsius) { return (celsius * 9.0 / 5.0) + 32.0; } // Максимум табу (varargs қолдану) public static int max(int... numbers) { if (numbers.length == 0) { throw new IllegalArgumentException("Кемінде бір сан беру керек"); } int maxValue = numbers[0]; for (int i = 1; i < numbers.length; i++) { if (numbers[i] > maxValue) { maxValue = numbers[i]; } } return maxValue; } // Факториал есептеу (рекурсивті static әдіс) public static long factorial(int n) { if (n < 0) { throw new IllegalArgumentException("Теріс сан болмауы керек"); } if (n <= 1) { return 1; } return n * factorial(n - 1); } // Static әдіс ішінде instance әдісін шақыра алмайды // private void instanceMethod() { } // Бұл әдісті static әдіс ішінде шақыра алмаймыз } // 2. String утилиталары class StringUtils { // String-ті керіге айналдыру public static String reverse(String str) { if (str == null || str.isEmpty()) { return str; } return new StringBuilder(str).reverse().toString(); } // Сөздер санын санау public static int countWords(String text) { if (text == null || text.trim().isEmpty()) { return 0; } return text.trim().split("\\s+").length; } // Palindrome тексеру public static boolean isPalindrome(String str) { if (str == null) return false; String cleaned = str.replaceAll("[^a-zA-ZА-Яа-яӘәІіҢңҒғҮүҰұҚқӨөҺһ]", "").toLowerCase(); return cleaned.equals(reverse(cleaned)); } } // Қолдану мысалы public class StaticMethodsDemo { public static void main(String[] args) { // MathUtils static әдістерін қолдану System.out.println("=== Математикалық операциялар ==="); System.out.println("5 + 3 = " + MathUtils.add(5, 3)); System.out.println("4 * 7 = " + MathUtils.multiply(4, 7)); System.out.println("Дөңгелек ауданы (r=5): " + MathUtils.circleArea(5.0)); System.out.println("25°C = " + MathUtils.celsiusToFahrenheit(25.0) + "°F"); System.out.println("Max(10, 5, 25, 15): " + MathUtils.max(10, 5, 25, 15)); System.out.println("5! = " + MathUtils.factorial(5)); // StringUtils static әдістерін қолдану System.out.println("\n=== String операциялары ==="); String text = "Сәлем, Java әлемі!"; System.out.println("Түпнұсқа: " + text); System.out.println("Кері: " + StringUtils.reverse(text)); System.out.println("Сөздер саны: " + StringUtils.countWords(text)); String palindrome = "мадам"; System.out.println("'" + palindrome + "' палиндром ма? " + StringUtils.isPalindrome(palindrome)); // Static тұрақтыларды қолдану System.out.println("\n=== Константалар ==="); System.out.println("π = " + MathUtils.PI); System.out.println("e = " + MathUtils.E); } }

⚠️ Static Әдістердің Шектеулері:

• Instance өрістер мен әдістерді тікелей қолдана алмайды
• this және super кілт сөздерін қолдана алмайды
• Instance әдістерден overriding болмайды
• Generic параметрлерді қолдану шектеулі

🏗️ Static Blocks (Инициализация Блоктары)

🔄 Static блоктардың орындалу реті:

1️⃣ Класс JVM-ға алғаш рет жүктелген кезде
2️⃣ Static өрістердің инициализациясынан кейін
3️⃣ Кез келген static әдіс шақырылғанға дейін
4️⃣ Кез келген объект құрылғанға дейін

🏗️ Static блоктардың мысалдары
// 1. Конфигурация класы class DatabaseConfig { // Static өрістер private static String url; private static String username; private static String password; private static boolean isConfigured = false; // Static блок - класс жүктелген кезде орындалады static { System.out.println("🔧 DatabaseConfig инициализациясы басталды..."); // Конфигурацияны жүктеу (симуляция) url = "jdbc:mysql://localhost:3306/mydb"; username = "admin"; password = "password123"; // Байланысты тексеру if (testConnection()) { isConfigured = true; System.out.println("✅ Дерекқор конфигурациясы сәтті жүктелді"); } else { System.out.println("❌ Дерекқор конфигурациясында қате"); } System.out.println("🏁 DatabaseConfig инициализациясы аяқталды"); } // Бірнеше static блок болуы мүмкін (ретімен орындалады) static { System.out.println("📊 Екінші static блок орындалуда..."); // Қосымша инициализация System.out.println("📈 Статистика жүйесі қосулды"); } // Static әдістер private static boolean testConnection() { // Байланысты тексеру симуляциясы try { Thread.sleep(100); // Байланыс уақытын имитациялау return true; } catch (InterruptedException e) { return false; } } public static String getConnectionString() { if (!isConfigured) { throw new RuntimeException("Дерекқор конфигурацияланбаған"); } return url + " (User: " + username + ")"; } } // 2. Математикалық константалар класы class MathConstants { public static final double PI; public static final double E; public static final double GOLDEN_RATIO; // Static блокта final өрістерді инициализациялау static { System.out.println("🔢 Математикалық константаларды есептеу..."); // PI есептеу (Лейбниц формуласы) double piCalculated = 0; for (int i = 0; i < 1000000; i++) { piCalculated += Math.pow(-1, i) / (2 * i + 1); } PI = 4 * piCalculated; // E есептеу (факториал арқылы) double eCalculated = 0; long factorial = 1; for (int i = 0; i < 20; i++) { if (i > 0) factorial *= i; eCalculated += 1.0 / factorial; } E = eCalculated; // Алтын қатынас GOLDEN_RATIO = (1 + Math.sqrt(5)) / 2; System.out.println("✅ Барлық константалар есептелді"); } public static void displayConstants() { System.out.println("π ≈ " + PI); System.out.println("e ≈ " + E); System.out.println("φ (Алтын қатынас) ≈ " + GOLDEN_RATIO); } } // Қолдану мысалы public class StaticBlockDemo { public static void main(String[] args) { System.out.println("=== Static блоктар демосы ==="); System.out.println("Main әдісі басталды\n"); // Алғашқы static әдіс шақыру - static блоктар орындалады System.out.println("DatabaseConfig класын қолдану:"); System.out.println(DatabaseConfig.getConnectionString()); System.out.println("\nМатематикалық константалар:"); MathConstants.displayConstants(); System.out.println("\n📝 Ескерту: Static блоктар тек бір рет орындалады!"); // Екінші рет шақырғанда static блоктар орындалмайды System.out.println(DatabaseConfig.getConnectionString()); } }

✨ Static Блоктардың артықшылықтары:

• Күрделі инициализация логикасын орындауға мүмкіндік береді
• Final static өрістерді есептеулер арқылы инициализациялауға болады
• Exception handling қолдануға болады
• Бір рет орындалуы ресурстарды үнемдейді

⚖️ Static vs Instance Салыстыру

Критерий Static Instance
Жадыдағы орны Method Area Heap Memory
Құрылу уақыты Класс жүктелген кезде new операторымен
Қол жетімділік КлассАты.мүше объект.мүше
Көптеген объектілер Барлығы үшін ортақ Әр объектіңде жеке
this, super қолдану ❌ Қолдана алмайды ✅ Қолдана алады
Overriding ❌ Болмайды ✅ Болады
Жады қолданысы Аз (бір данада) Көп (әр объектіде)
Пайдалану мақсаты Утилита әдістер, константалар Объектінің күйі мен әрекеттері

• Конфигурация мәліметтері үшін

🏭 Singleton Pattern

class DatabaseConnection {
  private static DatabaseConnection instance;
  private DatabaseConnection() {}

  public static DatabaseConnection getInstance() {
    if (instance == null) {
      instance = new DatabaseConnection();
    }
    return instance;
  }
}

💡 Static Қолдану Кеңестері

🎯 Дұрыс Қолдану

• Утилита әдістері үшін қолданыңыз
• Константалар үшін static final пайдаланыңыз
• Factory pattern-де қолданыңыз
• Сингleton паттернінде пайдаланыңыз

⚠️ Сақ Болу Керек

• Static өрістерді көп қолданбаңыз
• Mutable static state-тен аулақ болыңыз
• Thread safety туралы ойлаңыз
• Memory leak-тің алдын алыңыз

🚫 Тыйым Салынған

• Static импорт көп қолданбау
• Instance әдістерді static етпеу
• Static блокта exception лақтырмау
• Mutable static collections пайдаланбау

✅ Жақсы Практика

• Static final константалар UPPER_CASE
• Static әдістер үшін нақты атаулар
• Documentation жазу
• Unit тестілер жасау

🧠 Жады Басқару

• Static өрістер Method Area-да сақталады
• Garbage Collection-нан өтпейді
• ClassLoader өшкенге дейін қалады
• Memory профилирование жасаңыз

🔒 Thread Safety

• Static өрістер үшін синхронизация
• Immutable объектілер қолданыңыз
• ConcurrentHashMap пайдаланыңыз
• Atomic операцияларды қолданыңыз

📚 Қорытынды және Негізгі Ұғымдар

🎯 Static туралы негізгі ұғымдар:

Static — Java-дағы кілт сөз, ол класс мүшелерін (өрістер, әдістер, блоктар) объект құрмай-ақ қолдануға мүмкіндік береді. Static мүшелер класстің өзіне тиесілі, жеке объектілерге емес.

✅ Static Артықшылықтары

  • Жады үнемдеу (бір данада сақталады)
  • Объект құрмай-ақ қол жетімді
  • Утилита әдістері үшін ыңғайлы
  • Константалар үшін керемет
  • Класс деңгейіндегі операциялар

⚠️ Static Кемшіліктері

  • OOP принциптерін бұзуы мүмкін
  • Thread safety мәселелері
  • Тестілеуді қиындатады
  • Memory leak қаупі
  • Inheritance шектеулері

🚨 Маңызды ескертулер:

• Static әдістер instance мүшелерін тікелей қолдана алмайды
• Static өрістер барлық объектілер арасында ортақ
• Static блоктар класс жүктелген кезде бір рет орындалады
• Static импорт көп қолданбау керек
• Thread safety туралы ойлау қажет

🎖️ Сіз енді біледі:

• Static өрістер мен әдістерді қалай құру керектігін
• Static блоктардың қалай жұмыс істейтінін
• Static элементтерді қашан қолдану керектігін
• Static пен instance арасындағы айырмашылықты
• Практикалық мысалдарда static қалай қолданылатынын

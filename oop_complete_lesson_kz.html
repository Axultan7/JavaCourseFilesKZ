Объектіге бағытталған программалау - бұл нақты дүниедегі заттарды объекттер арқылы модельдеу тәсілі

DRY принципі - Don't Repeat Yourself

🎯 OOP-тың негізгі идеясы:

Программаны нақты дүниедегі объекттердің өзара әрекеттесуі ретінде қарастыру. Мысалы: Student объектісі Course объектісіне enrollment жасайды, Car объектісі Engine объектісін басқарады.

🏛️ OOP-тың 4 негізгі бағаны

Барлық объектіге бағытталған тілдер осы 4 принципке негізделген

🎁 Encapsulation

Инкапсуляция

Деректер мен методтарды бір жерде жинау және жасыру

Мысал: BankAccount класындағы balance field-ын private етіп, тек getter/setter арқылы қол жеткізуге рұқсат беру

👨‍👩‍👧‍👦 Inheritance

Мұрагерлік

Жаңа класстарды бар класстардан тудыру

Мысал: Student класы Person класынан мұра алады. Person-дағы name, age алады, өзіне studentId қосады

🎭 Polymorphism

Полиморфизм

Бір interface арқылы әртүрлі объекттермен жұмыс

Мысал: Animal[] массивінде Dog, Cat, Bird объекттері бар. Барлығы makeSound() методын әртүрлі іске асырады

🎨 Abstraction

Абстракция

Күрделілікті жасыру, тек қажетті интерфейсті көрсету

Мысал: Car класында drive() методы бар, бірақ двигатель қалай жұмыс істейтінін білудің қажеті жоқ

🎁 Encapsulation (Инкапсуляция)

Деректер мен олармен жұмыс істейтін методтарды бір жерде жинап, сыртқы әлемнен жасыру

💡 Encapsulation неліктен маңызды?

🔒 Қауіпсіздік: Маңызды деректерді кездейсоқ өзгертуден қорғау

✅ Бақылау: Деректерді өзгерту үрдісін бақылау (validation)

🧰 Ыңғайлылық: Қолданушыға тек қажетті функцияларды ұсыну

🔄 Икемділік: Ішкі іске асыруды өзгертіп, сыртқы интерфейсті сақтау

🎁 Encapsulation мысалы - Email класы
public class Email { // Private fields - сыртқы әлемнен жасырылған private String address; private boolean isValid; private String domain; // Constructor - validation бар public Email(String address) { setAddress(address); // Validation арқылы орнатамыз } // Public методтар - бақыланған қол жеткізу public String getAddress() { return address; } public void setAddress(String address) { if (isValidEmail(address)) { this.address = address; this.isValid = true; this.domain = extractDomain(address); } else { throw new IllegalArgumentException("Қате email форматы"); } } public boolean isValid() { return isValid; } public String getDomain() { return domain; } // Private helper методтар - ішкі логика private boolean isValidEmail(String email) { return email != null && email.contains("@") && email.contains("."); } private String extractDomain(String email) { return email.substring(email.indexOf("@") + 1); } }

✅ Encapsulation артықшылықтары:

• Қауіпсіздік: email-ді тек дұрыс форматта орнатуға болады

• Автоматтандыру: domain автоматты түрде есептеледі

• Икемділік: validation логикасын өзгертуге болады

• Оңай қолдану: кешенді логика жасырылған

👨‍👩‍👧‍👦 Inheritance (Мұрагерлік)

Жаңа класстарды бар класстардың негізінде жасау арқылы кодты қайта пайдалану

🌳 Мұрагерлік иерархиясы

⬇️
Student

Teacher

👨‍👩‍👧‍👦 Inheritance мысалы
// Негізгі класс (Parent/Base/Super class) public class Person { protected String name; // protected - subclass-тар көре алады protected int age; protected String address; public Person(String name, int age) { this.name = name; this.age = age; } public void introduce() { System.out.println("Менің атым " + name + ", жасым " + age); } public void walk() { System.out.println(name + " серуендеп жүр"); } } // Туынды класс (Child/Derived/Sub class) public class Student extends Person { private String studentId; private double gpa; private String university; // Constructor - super() арқылы parent constructor шақырамыз public Student(String name, int age, String studentId) { super(name, age); // Person конструкторын шақыру this.studentId = studentId; this.gpa = 0.0; } // Жаңа методтар - тек Student-қа тиесілі public void study(String subject) { System.out.println(name + " " + subject + " пәнін оқып жатыр"); } public void takeExam() { System.out.println(name + " емтихан тапсырып жатыр"); } // Method Override - parent методын қайта анықтау @Override public void introduce() { System.out.println("Мен студентпін. Атым " + name + ", студенттік нөмірім " + studentId); } // Getters/Setters public String getStudentId() { return studentId; } public double getGpa() { return gpa; } public void setGpa(double gpa) { this.gpa = gpa; } } // Басқа туынды класс public class Teacher extends Person { private String subject; private double salary; public Teacher(String name, int age, String subject) { super(name, age); this.subject = subject; } public void teach() { System.out.println(name + " " + subject + " сабағын өткізуде"); } @Override public void introduce() { System.out.println("Мен мұғаліммін. Атым " + name + ", " + subject + " пәнінен сабақ беремін"); } }

💡 Inheritance артықшылықтары:

• Код қайта пайдалану: Person-дағы кодты Student, Teacher қайта жазбайды

• Ұйымдастыру: Ортақ қасиеттерді бір жерде сақтау

• Кеңейту: Жаңа функционалдықты оңай қосу

• Полиморфизм: Person[] массивінде Student, Teacher сақтау

🎭 Polymorphism (Полиморфизм)

"Көп формалылық" - бір интерфейс арқылы әртүрлі типтегі объекттермен жұмыс істеу

🎯 Polymorphism-ның 2 түрі:

📝 Compile-time (Overloading)

Бір класста бірдей атаулы, бірақ әртүрлі параметрлі методтар

🏃 Runtime (Overriding)

Subclass-та parent методын қайта анықтау

🎭 Polymorphism мысалы - Animal иерархиясы
// Базалық класс abstract class Animal { protected String name; public Animal(String name) { this.name = name; } // Abstract метод - міндетті түрде override болуы керек public abstract void makeSound(); // Ортақ метод public void sleep() { System.out.println(name + " ұйықтап жатыр"); } } class Dog extends Animal { public Dog(String name) { super(name); } @Override public void makeSound() { System.out.println(name + " дауыстайды: Ауў!"); } // Dog-қа тән ерекше метод public void wagTail() { System.out.println(name + " құйрығын бұлғайды"); } } class Cat extends Animal { public Cat(String name) { super(name); } @Override public void makeSound() { System.out.println(name + " дауыстайды: Мяў!"); } public void purr() { System.out.println(name + " мырылдайды"); } } class Bird extends Animal { public Bird(String name) { super(name); } @Override public void makeSound() { System.out.println(name + " дауыстайды: Чирик!"); } public void fly() { System.out.println(name + " ұшып жүр"); } } // Polymorphism-ды қолдану public class AnimalDemo { public static void main(String[] args) { // Polymorphism - Animal reference-пен әртүрлі объекттер Animal[] animals = { new Dog("Рекс"), new Cat("Мурка"), new Bird("Кеша"), new Dog("Бобик") }; // Polymorphic метод шақыру System.out.println("=== Барлық жануарлар дауыстайды ==="); for (Animal animal : animals) { animal.makeSound(); // Әр объект өз makeSound() методын орындайды } // instanceof арқылы нақты типті тексеру System.out.println("\n=== Ерекше әрекеттер ==="); for (Animal animal : animals) { if (animal instanceof Dog) { ((Dog) animal).wagTail(); // Type casting } else if (animal instanceof Cat) { ((Cat) animal).purr(); } else if (animal instanceof Bird) { ((Bird) animal).fly(); } } } }

✅ Polymorphism артықшылықтары:

• Икемділік: Жаңа Animal түрлерін оңай қосуға болады

• Жалпылау: Animal[] массивінде кез келген жануарды сақтау

• Кодтың азаюы: әр типке жеке код жазудың қажеті жоқ

• Кеңейту: жаңа субкласстар қосқанда негізгі код өзгермейді

🎨 Abstraction (Абстракция)

Күрделі ішкі құрылымды жасырып, тек қажетті функционалдықты көрсету

🎯 Abstraction-ның 2 тәсілі:

🏗️ Abstract класстар

Кейбір методтар abstract, кейбіреуі конкретті іске асырылған

📋 Interface-тар

Тек method signatures бар, іске асыру жоқ (Java 8-ге дейін)

🎨 Abstraction мысалы - Көлік жүйесі
// Abstract класс - толық іске асырылмаған abstract class Vehicle { protected String brand; protected double speed; protected boolean isRunning; public Vehicle(String brand) { this.brand = brand; this.speed = 0; this.isRunning = false; } // Конкретті методтар - барлық субкласстар пайдалана алады public void start() { isRunning = true; System.out.println(brand + " іске қосылды"); } public void stop() { isRunning = false; speed = 0; System.out.println(brand + " тоқтады"); } // Abstract методтар - міндетті түрде іске асыру керек public abstract void accelerate(); public abstract void brake(); public abstract int getMaxSpeed(); } // Interface - толығымен abstract interface Flyable { void takeOff(); void land(); int getAltitude(); } // Concrete класс class Car extends Vehicle { private int doors; public Car(String brand, int doors) { super(brand); this.doors = doors; } @Override public void accelerate() { if (isRunning) { speed += 10; System.out.println(brand + " жеделдетілді. Жылдамдық: " + speed); } } @Override public void brake() { speed = Math.max(0, speed - 15); System.out.println(brand + " тежелді. Жылдамдық: " + speed); } @Override public int getMaxSpeed() { return 180; } } // Interface-ті іске асыратын класс class Airplane extends Vehicle implements Flyable { private int altitude; public Airplane(String brand) { super(brand); this.altitude = 0; } @Override public void accelerate() { if (isRunning) { speed += 50; System.out.println(brand + " жеделдетілді. Жылдамдық: " + speed); } } @Override public void brake() { speed = Math.max(0, speed - 30); System.out.println(brand + " баяулатылды. Жылдамдық: " + speed); } @Override public int getMaxSpeed() { return 900; } // Flyable interface методтарын іске асыру @Override public void takeOff() { if (speed > 200) { altitude = 1000; System.out.println(brand + " ұшып кетті!"); } } @Override public void land() { altitude = 0; System.out.println(brand + " жерге қонды"); } @Override public int getAltitude() { return altitude; } }

💡 Abstract vs Interface айырмашылығы:

• Abstract класс: Конкретті және abstract методтар бір аралас болуы мүмкін

• Interface: Тек method signatures (Java 8-ден кейін default/static методтар бар)

• Мұрагерлік: Abstract-тан тек бір іске, interface-тен көп іске асыруға болады

• Қолдану: Abstract - "is-a" қатынас, Interface - "can-do" мүмкіндік

⚖️ OOP vs Procedural Programming

Объектіге бағытталған және процедуралық программалау парадигмаларын салыстыру

📜 Procedural Programming

Негізгі идея: Функциялар мен процедуралар
Артықшылықтары:
  • Қарапайым және түсінікті
  • Кішкене програмалар үшін жылдам
  • Азырақ жады қолданады
  • Математикалық есептеулер үшін ыңғайлы
Кемшіліктері:
  • Үлкен проекттерде басқару қиын
  • Кодты қайта пайдалану қиын
  • Global деректерге тәуелділік
  • Сәйкестендіру және тестілеу қиын

🏛️ Object-Oriented Programming

Негізгі идея: Объекттер мен олардың өзара әрекеттесуі
Артықшылықтары:
  • Модульділік және ұйымдастыру
  • Кодты қайта пайдалану (inheritance)
  • Encapsulation арқылы қауіпсіздік
  • Үлкен проекттерді басқару оңай
  • Команда жұмысы үшін ыңғайлы
Кемшіліктері:
  • Бастапқыда күрделі көрінеді
  • Көбірек жады қолданады
  • Кішкене программалар үшін артық
  • Үйрену қиынырақ
⚖️ Салыстыру мысалы - калькулятор
// PROCEDURAL STYLE - функциялар негізінде public class ProceduralCalculator { public static double add(double a, double b) { return a + b; } public static double subtract(double a, double b) { return a - b; } public static void main(String[] args) { double result = add(5, 3); System.out.println("Нәтиже: " + result); } } // OOP STYLE - объекттер негізінде public class OOPCalculator { private double memory; private String lastOperation; public OOPCalculator() { this.memory = 0; this.lastOperation = "Ешнәрсе"; } public double add(double a, double b) { double result = a + b; memory = result; lastOperation = a + " + " + b; return result; } public double subtract(double a, double b) { double result = a - b; memory = result; lastOperation = a + " - " + b; return result; } public void showHistory() { System.out.println("Соңғы операция: " + lastOperation); System.out.println("Жадында: " + memory); } public static void main(String[] args) { OOPCalculator calc = new OOPCalculator(); double result = calc.add(5, 3); calc.showHistory(); } }

🌟 OOP-тың артықшылықтары

Неліктен заманауи программалауда OOP басым?

🧱 Модульділік

Кодты кіші бөліктерге бөлу

Әр класс өз жауапкершілігіне ие. Бір бөлік бұзылса, басқалары жұмыс істей береді.

Separation of Concerns принципі

🔄 Қайта пайдалану

Бір рет жаз, көп жерде қолдан

Inheritance арқылы бар кодты жаңа класстарда қайта пайдалану. Библиотекалар жасау.

DRY - Don't Repeat Yourself

🛡️ Қауіпсіздік

Деректерді қорғау

Encapsulation арқылы маңызды деректерді жасыру. Тек рұқсат етілген операциялар.

Data integrity және security

🔧 Қызмет көрсету

Кодты өзгерту оңай

Бір класстағы өзгерістер басқа класстарға әсер етпейді. Баг табу оңай.

Maintainability және debugging

📈 Масштабтау

Үлкен проекттер үшін

Мыңдаған жол кодпен жұмыс істеу. Команда жұмысы. Enterprise деңгейдегі жүйелер.

Scalability және teamwork

🎭 Икемділік

Polymorphism мүмкіндіктері

Бір интерфейс арқылы әртүрлі объекттермен жұмыс. Жаңа типтерді оңай қосу.

Flexibility және extensibility

🎯 Нақтылық

Нақты дүниені модельдеу

Программа құрылымы нақты қолданба аясымен сәйкес келеді. Түсінуге оңай.

Real-world modeling

🧪 Тестілеу

Unit testing мүмкіндігі

Әр класстың функционалдығын жеке тексеру. Automated testing жүйелері.

Testability және quality assurance

🌍 Нақты өмірдегі OOP мысалдар

OOP принциптері қайда қолданылады?

🏦 Банк жүйесі

Класстар: Account, Customer, Transaction, CreditCard
Encapsulation: Balance қорғалған, тек методтар арқылы
Inheritance: SavingsAccount < Account, CheckingAccount < Account
Polymorphism: Әртүрлі account типтері бірдей интерфейс
Abstraction: User транзакция детальдарын білмейді

🎮 Ойын әзірлеу

Класстар: Player, Enemy, Weapon, Level, Game
Encapsulation: Player health private, getter/setter арқылы
Inheritance: Warrior < Player, Archer < Player
Polymorphism: Барлық character бірдей attack() методын орындайды
Abstraction: Input handling, graphics жасырылған

🛒 E-commerce платформа

Класстар: Product, Customer, Order, ShoppingCart, Payment
Encapsulation: Price, inventory count қорғалған
Inheritance: Electronics < Product, Clothing < Product
Polymorphism: Әртүрлі payment методтары
Abstraction: Database operations жасырылған

🏥 Медицина жүйесі

Класстар: Patient, Doctor, MedicalRecord, Appointment
Encapsulation: Medical data құпия, role-based access
Inheritance: Surgeon < Doctor, Nurse < MedicalStaff
Polymorphism: Әртүрлі treatment типтері
Abstraction: Prescription logic жасырылған

🚗 Көлік басқару

Класстар: Vehicle, Engine, GPS, FuelTank, Dashboard
Encapsulation: Engine parameters қорғалған
Inheritance: Car < Vehicle, Motorcycle < Vehicle
Polymorphism: Әртүрлі vehicle типтері бірдей start()
Abstraction: ECU, transmission logic жасырылған

📱 Мобильді қосымшалар

Класстар: User, Activity, Fragment, Database, API
Encapsulation: User credentials қорғалған
Inheritance: MainActivity < Activity
Polymorphism: Әртүрлі View типтері
Abstraction: Network operations, threading жасырылған

💡 OOP жақсы тәжірибелері

🎯 SOLID принциптері

S - Single Responsibility: Бір класс - бір жауапкершілік

O - Open/Closed: Кеңейтуге ашық, өзгертуге жабық

L - Liskov Substitution: Subclass parent орнын ауыстыра алуы керек

I - Interface Segregation: Кіші, мамандандырылған interface-тар

D - Dependency Inversion: Абстракцияға тәуелді болу

📝 Атау конвенциялары

Класстар: PascalCase (Student, BankAccount)

Методтар: camelCase (getName, calculateTotal)

Айнымалылар: camelCase (firstName, maxValue)

Константалар: UPPER_CASE (MAX_SIZE, PI)

Interface: I-мен басталу (IPayable, IComparable)

🏗️ Дизайн принциптері

Composition > Inheritance: Мұрагерлікке қарағанда композиция дұрысырақ

Program to interfaces: Конкретті класстарға емес, интерфейстарға

Favor object composition: Объекттерді біріктіру арқылы функционалдық

Encapsulate what varies: Өзгеретін бөліктерді жасыру

🔒 Encapsulation ережелері

Fields: Әрқашан private, рұқсат етілген жағдайдан басқа

Getters: Қарапайым, қосымша логика жоқ

Setters: Validation логикасымен

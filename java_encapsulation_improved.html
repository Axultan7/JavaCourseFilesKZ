Деректерді қорғау және инкапсуляция принциптері

Encapsulation — деректерді жасыру және қауіпсіз қол жетімділік

🎯 Encapsulation дегеніміз не?

📖 Анықтама

Encapsulation (инкапсуляция) — объектіге бағытталған программалаудың (OOP) негізгі принциптерінің бірі. Ол деректер мен кодты бір орында біріктіріп, сыртқы кедергілерден қорғайды. Деректерге тікелей қол жеткізуді шектеп, арнайы әдістер арқылы басқаруды қамтамасыз етеді.

Код өзгертуді жеңілдетеді

🔑 Access Modifiers (Қол жетімділік модификаторлары)

🎯 Төрт деңгейлі қол жетімділік

Java-да төрт түрлі қол жетімділік деңгейі бар. Олар деректер мен әдістерге кімнің қол жеткізе алатынын анықтайды. Дұрыс модификатор таңдау — қауіпсіз код жазудың негізі.

🚫 Дұрыс Емес vs ✅ Дұрыс Encapsulation

❌ Дұрыс ЕМЕС (Encapsulation жоқ)

💀 Қауіпті код
class BadStudent { // Public өрістер - қауіпті! public String name; public int age; public double gpa; } // Қолдану: BadStudent student = new BadStudent(); student.age = -5; // Қате мән! student.gpa = 150.0; // Мүмкін емес! student.name = ""; // Бос мән!
Мәселелер:
• Деректерді тексерусіз өзгертуге болады
• Қате мәндер беруге болады
• Код қауіпсіз емес
• Логика бақылауы жоқ

✅ ДҰРЫС (Encapsulation бар)

🛡️ Қауіпсіз код
class GoodStudent { // Private өрістер - қауіпсіз! private String name; private int age; private double gpa; // Getter/Setter арқылы бақылау public void setAge(int age) { if (age > 0 && age < 150) { this.age = age; } else { throw new IllegalArgumentException( "Жас дұрыс емес!"); } } public int getAge() { return age; } }
Артықшылықтар:
• Деректер қорғалған
• Валидация бар
• Бақылаулы қол жетімділік
• Қауіпсіз код

👤 Толық Encapsulation Мысалы

🔒 Толығымен инкапсуляцияланған класс
// Тұлға туралы толық мәліметтер класы class Person { // Private өрістер - тікелей қол жетімділік жоқ private String firstName; private String lastName; private int age; private String email; private String phoneNumber; private double salary; // Constructor - объект құру public Person(String firstName, String lastName, int age) { setFirstName(firstName); setLastName(lastName); setAge(age); } // Getter әдістері - деректерді оқу public String getFirstName() { return firstName; } public String getLastName() { return lastName; } public String getFullName() { return firstName + " " + lastName; } public int getAge() { return age; } public String getEmail() { return email; } public String getPhoneNumber() { return phoneNumber; } // Salary-ді жасырын ұстау - тек белгілі шарттарда көрсету public String getSalaryInfo(boolean isAuthorized) { if (isAuthorized) { return "Жалақы: " + salary + " тг"; } else { return "Жалақы туралы ақпарат жабық"; } } // Setter әдістері - деректерді жазу (валидациямен) public void setFirstName(String firstName) { if (firstName != null && !firstName.trim().isEmpty()) { // Тек әріптерді қабылдау if (firstName.matches("[a-zA-ZА-Яа-яӘәІіҢңҒғҮүҰұҚқӨөҺһ\\s]+")) { this.firstName = firstName.trim(); } else { throw new IllegalArgumentException("Аты тек әріптерден тұруы керек"); } } else { throw new IllegalArgumentException("Аты бос болмауы керек"); } } public void setLastName(String lastName) { if (lastName != null && !lastName.trim().isEmpty()) { if (lastName.matches("[a-zA-ZА-Яа-яӘәІіҢңҒғҮүҰұҚқӨөҺһ\\s]+")) { this.lastName = lastName.trim(); } else { throw new IllegalArgumentException("Тегі тек әріптерден тұруы керек"); } } else { throw new IllegalArgumentException("Тегі бос болмауы керек"); } } public void setAge(int age) { if (age >= 0 && age <= 150) { this.age = age; } else { throw new IllegalArgumentException("Жас 0-150 аралығында болуы керек"); } } public void setEmail(String email) { if (email != null && isValidEmail(email)) { this.email = email.toLowerCase(); } else { throw new IllegalArgumentException("Email форматы дұрыс емес"); } } public void setSalary(double salary, boolean isAuthorized) { if (!isAuthorized) { throw new IllegalAccessError("Жалақы өзгерту үшін рұқсат жоқ"); } if (salary >= 0) { this.salary = salary; } else { throw new IllegalArgumentException("Жалақы теріс болмауы керек"); } } // Private көмекші әдістер - валидация private boolean isValidEmail(String email) { // Жеңілдетілген email валидациясы return email.matches("^[A-Za-z0-9+_.-]+@[A-Za-z0-9.-]+\\.[A-Za-z]{2,}$"); } }

💡 Encapsulation-дың негізгі ерекшеліктері:

• Private өрістер: Деректерге тікелей қол жетімділікті шектейді
• Public методтар: Бақылаулы қол жетімділік ұсынады
• Валидация: Setter әдістерінде деректерді тексереді
• Қауіпсіздік: Авторизация және рұқсат тексеру
• Форматтау: Деректерді стандартты түрге келтіреді

🔄 Getter және Setter Әдістері

🎯 Getter/Setter құрылымы

Getter: private өрістің мәнін оқу үшін қолданылады
Setter: private өрістің мәнін орнату үшін қолданылады
Валидация: setter-де деректерді тексеру мүмкіндігі
Форматтау: деректерді дұрыс түрде сақтау

🎓 Студент класының Getter/Setter мысалы
// Студент класы - толық encapsulation class Student { // Private өрістер private String studentId; private String name; private int age; private double gpa; private String major; private int coursesCompleted; // Constructor public Student(String name, int age, String major) { setName(name); setAge(age); setMajor(major); this.gpa = 0.0; this.coursesCompleted = 0; this.studentId = generateStudentId(); } // Getter әдістері public String getStudentId() { return studentId; } public String getName() { return name; } public int getAge() { return age; } public double getGpa() { return Math.round(gpa * 100.0) / 100.0; // 2 ондық үтір } // Есептелетін properties public String getAcademicLevel() { if (coursesCompleted < 30) return "1-курс"; else if (coursesCompleted < 60) return "2-курс"; else if (coursesCompleted < 90) return "3-курс"; else return "4-курс"; } // Setter әдістері (валидациямен) public void setName(String name) { if (name == null || name.trim().isEmpty()) { throw new IllegalArgumentException("Студент аты бос болмауы керек"); } this.name = name.trim(); } private String generateStudentId() { return "STU" + System.currentTimeMillis() % 100000; } }

✨ Getter/Setter әдістерінің артықшылықтары:

• Қауіпсіздік: Private өрістерге бақылаулы қол жетімділік
• Валидация: Setter-де деректерді тексеру мүмкіндігі
• Есептелетін properties: Динамикалық мәндерді қайтару
• Форматтау: Деректерді дұрыс түрде ұсыну
• Логика қосу: Қосымша бизнес логика енгізу

🔐 Банк шоты - Қауіпсіздік мысалы

🛡️ Валидацияның түрлері

Мән тексеру: null, бос жолдар, диапазон
Формат тексеру: email, телефон, дата
Бизнес ережелер: мамандандырылған логика
Қауіпсіздік: авторизация, рұқсат тексеру

🏦 Банк шоты класы - жоғары деңгейлі қауіпсіздік
// Банк шоты - жоғары деңгейлі қауіпсіздік class BankAccount { // Private өрістер private final String accountNumber; private String accountHolderName; private double balance; private String pin; private boolean isActive; private int failedAttempts; private final double MIN_BALANCE = 1000.0; private final int MAX_FAILED_ATTEMPTS = 3; // Constructor public BankAccount(String accountHolderName, double initialDeposit, String pin) { this.accountNumber = generateAccountNumber(); setAccountHolderName(accountHolderName); setPin(pin); setInitialBalance(initialDeposit); this.isActive = true; this.failedAttempts = 0; } // Балансты тек PIN арқылы көруге болады public double getBalance(String inputPin) { if (!validatePin(inputPin)) { throw new SecurityException("Қате PIN код"); } return balance; } // Ақша салу - валидациямен public void deposit(double amount, String pin) { if (!validatePin(pin)) { throw new SecurityException("Қате PIN код"); } if (amount <= 0) { throw new IllegalArgumentException("Салым мөлшері оң сан болуы керек"); } balance += amount; System.out.println("💰 Салым: " + amount + " тг. Жаңа баланс: " + balance + " тг"); } // Private көмекші әдістер private boolean validatePin(String inputPin) { if (!isActive) { throw new SecurityException("Шот блокталған"); } if (inputPin == null || !inputPin.equals(pin)) { failedAttempts++; if (failedAttempts >= MAX_FAILED_ATTEMPTS) { isActive = false; throw new SecurityException("Шот блокталды! Көп рет қате PIN енгізілді"); } return false; } failedAttempts = 0; // Сәтті енгізу return true; } }

🔒 Қауіпсіздік ережелері:

• Сезімтал деректерді (пароль, PIN) ешқашан public етпеңіз
• Барлық кіру мәліметтерін валидациядан өткізіңіз
• Қате әрекеттерді санап, шот блоктауды енгізіңіз
• Қауіпсіз әдістер арқылы ғана деректерге қол жеткізіңіз
• Exception handling арқылы қатені дұрыс өңдеңіз
